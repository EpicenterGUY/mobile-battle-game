# SKILL_REWORK_PLAN

## 문서 목적
본 문서는 **코드 변경 없이** 현재 스킬 시스템의 단조로움/역할 중복 문제를 해결하기 위한 전체 리워크 설계안이다.  
대상은 `main.js`, `data.js` 전투 로직을 중심으로 한 스킬 구조이며, 이번 단계에서는 구현이 아닌 **설계 원칙/우선순위/리스크 관리**를 정의한다.

---

## 1) 현재 문제점

### 1-1. 위력만 다른 공격 스킬 다수
- 현재 스킬 간 차이가 `power` 수치 중심이라, 플레이 감각이 "강한 숫자 고르기"로 수렴.
- 결과적으로 스킬 슬롯 다양성이 있어도 전투 의사결정 다양성은 낮음.

### 1-2. 공격 감소 수치만 다른 약화 스킬 다수
- 디버프 계열이 "감소량만 다른 같은 효과"로 느껴짐.
- 상황(선제/역전/마무리)에 따른 용도 분리가 약함.

### 1-3. 방어율만 다른 방어기 다수
- 생존 스킬이 모두 유사한 타이밍(위기 시 사용)으로 몰림.
- 탱킹/버티기/반격 준비/해제 대응 같은 역할 분화가 부족.

### 1-4. 스킬 선택 이유 약함
- 스킬 선택 UI는 있으나 "왜 이 스킬을 지금 써야 하는가"를 만드는 시스템적 보상이 적음.
- 상태이상, 카운터, 궁극기 게이지 연계가 약해 빌드 아이덴티티가 흐려짐.

---

## 2) 스킬 설계 원칙

모든 스킬은 아래 요소 중 **최소 1개 이상**, 가능하면 **2~3개 조합**을 가져야 한다.

1. **조건부 효과**: 특정 조건에서 추가 효과 발동 (예: 표식 대상에게 추가타)
2. **리스크**: 강력하지만 비용/약점 존재 (예: 반동, 자가 취약)
3. **콤보**: 선행 스킬이 후속 스킬 가치를 올림
4. **상태이상**: 단순 데미지 외 전장 상태를 바꿈
5. **카운터 역할**: 특정 적 유형/버프/패턴을 견제
6. **궁극기 게이지 운영**: 빠른 충전 vs 고효율 소비 선택지 제공
7. **로테이션 연계**: 2~4턴 순환 운용이 유리하도록 설계
8. **서브스킬 시너지**: 액티브-패시브(또는 보조 액티브) 상호작용
9. **마무리 역할**: 처형/체력 조건 마감 능력
10. **생존 역할**: 단순 방어 외 보호막/정화/피해 분산 등 차별화

### 설계 규칙(권장)
- "숫자 차이" 대신 "행동 결과 차이"를 만든다.
- 동일 계열 스킬은 사용 타이밍이 겹치지 않게 목적을 분리한다.
- 고성능 스킬에는 쿨다운, 조건, 리스크 중 최소 1개 부여.
- PvE/PvP(온라인) 모두에서 악용 가능성을 선제 점검.

## 3) 신규 상태효과 후보 정의

아래 상태효과는 추후 `effects` 구조로 모델링 가능한 기준으로 정의한다.

### 3-1. 표식 (Mark)
- **효과 설명**: 대상에게 표식을 남겨, 특정 스킬의 추가 피해/부가 효과 트리거.
- **지속 방식**: 턴 기반(예: 2턴) 또는 1회 소모형.
- **해제 조건**: 표식 연계 스킬 적중 시 소모, 정화로 해제 가능.
- **어울리는 캐릭터**: 궁수, 도적, 검사.
- **밸런스 주의점**: 표식+고배율 스킬 폭딜 상한 관리(치명타 중첩 포함).

### 3-2. 취약 (Vulnerable)
- **효과 설명**: 받는 피해 증가(최종 피해 배율 상승).
- **지속 방식**: 짧은 턴(1~2턴) 권장.
- **해제 조건**: 턴 종료/정화/보호막 획득 시 약화(선택 규칙).
- **어울리는 캐릭터**: 마법사, 광전사, 주술사.
- **밸런스 주의점**: 다중 취약 중첩 금지 또는 감쇠 적용 필수.

### 3-3. 출혈 (Bleed)
- **효과 설명**: 턴 종료 시 고정 또는 공격력 계수 기반 지속 피해.
- **지속 방식**: 2~4턴, 중첩 가능 시 최대 스택 제한.
- **해제 조건**: 정화, 특정 힐/방어 스킬로 완화.
- **어울리는 캐릭터**: 도적, 광전사, 검사.
- **밸런스 주의점**: 고속 타수 캐릭터와 결합 시 누적 폭주 방지.

### 3-4. 보호막 (Shield)
- **효과 설명**: 일정량 피해를 체력 대신 우선 흡수.
- **지속 방식**: 수치 소진형 + 턴 제한(예: 2턴).
- **해제 조건**: 흡수량 소진, 방어해제에 의한 제거.
- **어울리는 캐릭터**: 탱커, 성직자.
- **밸런스 주의점**: 힐+보호막 무한 유지 루프 차단(갱신 페널티 고려).

### 3-5. 집중 (Focus)
- **효과 설명**: 다음 공격/스킬의 효율 상승(명중/치명/관통/위력 중 1~2개).
- **지속 방식**: 다음 공격 1회 소모형.
- **해제 조건**: 공격 실행, 턴 경과, 방해 디버프.
- **어울리는 캐릭터**: 검사, 궁수, 마법사.
- **밸런스 주의점**: 선턴 원콤 방지 위해 증폭 상한 필요.

### 3-6. 방어관통 (Armor Pierce)
- **효과 설명**: 대상 방어율 일부 무시.
- **지속 방식**: 단일 타격 부여형 또는 1턴 버프형.
- **해제 조건**: 공격 후 소모.
- **어울리는 캐릭터**: 검사, 광전사, 궁수.
- **밸런스 주의점**: 탱커 무력화가 과도하지 않도록 최소 보정선 유지.

### 3-7. 방어해제 (Dispel Guard)
- **효과 설명**: 대상의 방어 버프/보호막/집중 등 이로운 효과 제거.
- **지속 방식**: 즉시 효과.
- **해제 조건**: 즉발형이므로 없음.
- **어울리는 캐릭터**: 주술사, 마법사, 성직자(정화의 역방향).
- **밸런스 주의점**: 핵심 생존기 완전 봉쇄 방지(제거 우선순위/제거 수 제한).

### 3-8. 궁극기 충전 (Ultimate Charge)
- **효과 설명**: 궁극기 게이지 획득량 증가 또는 즉시 충전.
- **지속 방식**: 턴 기반 버프(획득량 증가) + 즉시 충전형 혼합 가능.
- **해제 조건**: 턴 종료, 궁극기 사용 시 소모.
- **어울리는 캐릭터**: 마법사, 성직자, 주술사.
- **밸런스 주의점**: 게이지 가속 메타 고착 방지(턴당/전투당 캡 필요).

### 3-9. 정화 (Cleanse)
- **효과 설명**: 아군 디버프 제거, 일부 상태 저항 부여.
- **지속 방식**: 즉시 해제 + 짧은 저항 버프(선택).
- **해제 조건**: 저항 버프는 턴 종료 시 소멸.
- **어울리는 캐릭터**: 성직자, 탱커, 주술사.
- **밸런스 주의점**: 디버프 메타 완전 무효화 방지(해제 가능 등급 구분 필요).

## 4) 스킬 데이터 구조 개선안

## 4-1. 기존 `type` 단일 방식의 한계
- **확장성 부족**: 스킬 하나에 다중 효과(피해+표식+자가리스크)를 표현하기 어려움.
- **조건식 난립**: 전투 로직에서 `if (type === ...)` 분기가 폭증.
- **재사용성 저하**: 유사 효과를 새 type으로 계속 복제하게 됨.
- **밸런싱 비용 증가**: 수치 조정 포인트가 분산되고 의도 추적이 어려움.

## 4-2. `effects[]` 배열 방식의 장점
- **조합형 설계**: 하나의 스킬에 여러 모듈형 효과를 선언적으로 결합 가능.
- **로직 단순화**: 효과 처리 파이프라인(순차 적용)으로 엔진 일관성 확보.
- **콘텐츠 생산성 향상**: 신규 스킬 제작 시 기존 effect 조합으로 빠르게 생성.
- **테스트 용이성**: effect 단위 테스트 가능(데미지/상태부여/해제 개별 검증).

예시:
```js
effects: [
  { type: "damage", power: 12 },
  { type: "applyPoison", turns: 3 }
]
```

권장 확장 예시:
```js
effects: [
  { type: "damage", power: 10, scaling: "atk" },
  { type: "applyStatus", status: "mark", turns: 2, chance: 0.8 },
  { type: "conditional", if: "targetHasMark", then: [{ type: "bonusDamage", power: 8 }] },
  { type: "selfStatus", status: "focus", stacks: 1 }
]
```

## 5) 캐릭터별 리워크 방향 (8종)

## A. 검사
- **핵심 컨셉**: 정밀한 1:1 결투, 표식-관통-마무리의 교과서형 딜러.
- **스킬 역할 분배**: 기본기(안정딜) / 표식 부여 / 관통 피니시 / 반격성 생존기.
- **추천 신규 효과**: 표식, 집중, 방어관통.
- **유지할 것**: 단일 대상 기본 공격형 스킬(정체성 유지).
- **리워크할 것**: 단순 고위력기 → 표식 소비형 피니시로 전환.
- **빌드 예시**:
  1) 표식 콤보형: 표식 부여 → 집중 → 관통 피니시
  2) 안정 결투형: 방어 자세 → 반격 트리거 → 체력 조건 마무리

## B. 마법사
- **핵심 컨셉**: 상태이상 증폭과 타이밍 폭발(취약/방어해제 중심).
- **스킬 역할 분배**: 광역 압박 / 취약 부여 / 해제 카운터 / 궁극기 가속.
- **추천 신규 효과**: 취약, 방어해제, 궁극기 충전.
- **유지할 것**: 광역 공격 스킬(직관적 재미).
- **리워크할 것**: 위력만 다른 화력기 → 상태 연동형 주문으로 통합.
- **빌드 예시**:
  1) 디버프 누적형: 취약 부여 → 방어해제 → 폭발 주문
  2) 궁극 순환형: 충전 버프 → 저비용 견제 → 궁극기 빠른 회전

## C. 도적
- **핵심 컨셉**: 출혈 누적과 빈틈 처벌, 고위험 고보상 암살.
- **스킬 역할 분배**: 연타 누적 / 출혈 부여 / 회피성 생존 / 처형기.
- **추천 신규 효과**: 출혈, 표식, 마무리 보정.
- **유지할 것**: 빠른 타격 계열.
- **리워크할 것**: 단순 연타 스킬 → 출혈 스택 기반 강화 구조.
- **빌드 예시**:
  1) 출혈 스택형: 다단히트 → 출혈 확대 → 처형
  2) 기동 암살형: 표식 → 은신성 버프(향후) → 고배율 일격

## D. 탱커
- **핵심 컨셉**: 피해 흡수와 아군 보호, 적 버스트 차단.
- **스킬 역할 분배**: 보호막 / 도발성 압박(향후) / 정화 보조 / 카운터 방어.
- **추천 신규 효과**: 보호막, 정화, 방어해제 저항.
- **유지할 것**: 방어 자세 스킬.
- **리워크할 것**: 방어율 숫자 차이 스킬군 → 보호막/피해분산 역할 분리.
- **빌드 예시**:
  1) 메인탱형: 대형 보호막 → 정화 보조 → 버스트 흡수
  2) 카운터탱형: 방어 태세 → 반격/약화 부여 → 교착 유도

## E. 광전사
- **핵심 컨셉**: 체력 리스크를 대미지로 환산하는 폭주 딜러.
- **스킬 역할 분배**: 자가 리스크 버프 / 관통 강타 / 출혈 확산 / 종결기.
- **추천 신규 효과**: 취약(자가/적), 출혈, 방어관통.
- **유지할 것**: 고위력 근접기.
- **리워크할 것**: 무조건 강한 일격 → 체력 조건/반동 조건부 일격.
- **빌드 예시**:
  1) 저체력 광폭형: 자가 취약 감수 → 관통 폭딜
  2) 출혈 난전형: 광역 출혈 → 스택 소모 폭발

## F. 성직자
- **핵심 컨셉**: 회복+정화+보호의 전장 유지 관리자.
- **스킬 역할 분배**: 단일 회복 / 광역 보조 / 정화 / 보호막.
- **추천 신규 효과**: 정화, 보호막, 궁극기 충전 보조.
- **유지할 것**: 기본 회복기.
- **리워크할 것**: 회복량만 다른 힐 스킬군 → 상황별(즉발/지속/정화 동반) 분화.
- **빌드 예시**:
  1) 안정 유지형: 보호막 선부여 → 지속 회복 → 정화 대응
  2) 궁극 지원형: 충전 보조 → 타이밍 힐 → 위기 궁극 연계

## G. 궁수
- **핵심 컨셉**: 표식 기반 정밀 사격과 거리감 있는 턴 설계.
- **스킬 역할 분배**: 표식 부여 / 집중 준비 / 관통 사격 / 마무리 저격.
- **추천 신규 효과**: 표식, 집중, 방어관통.
- **유지할 것**: 단일 고정밀 공격기.
- **리워크할 것**: 단순 연사/강사 구분 → 표식 소비/비소비 분기.
- **빌드 예시**:
  1) 저격형: 표식 2중첩 → 집중 → 마무리 샷
  2) 압박형: 관통 견제 → 재표식 → 쿨순환 딜

## H. 주술사
- **핵심 컨셉**: 약화/해제/교란을 통한 전장 컨트롤.
- **스킬 역할 분배**: 디버프 부여 / 버프 제거 / 저항 저하 / 보조 충전.
- **추천 신규 효과**: 취약, 방어해제, 정화(아군)/저주(적) 양면 운용.
- **유지할 것**: 디버프형 스킬 정체성.
- **리워크할 것**: 감소량만 다른 약화기 → 역할별 저주 패키지화.
- **빌드 예시**:
  1) 해제 카운터형: 적 버프 제거 → 취약 부여 → 아군 딜 타이밍 제공
  2) 교란 유지형: 디버프 순환 → 적 로테이션 붕괴 유도

---

## 6) 1차 구현 우선순위 (안전 순서)

1. **신규 상태효과 3~5개만 먼저 추가**
   - 권장 스타트: `표식`, `취약`, `보호막`, `정화` (+ 선택 `출혈`)
2. **검사/마법사만 먼저 리워크**
   - 단일 타겟 콤보형(검사) + 상태 증폭형(마법사)으로 대비 실험
3. **테스트 후 나머지 캐릭터 순차 리워크**
   - 도적/탱커 → 광전사/성직자 → 궁수/주술사
4. **마지막에 밸런스 조정**
   - 승률, 평균 턴수, 궁극 사용 빈도, 상태효과 기여도 기준으로 미세조정

## 7) 구현 리스크

1. **온라인 동기화**
   - 상태 스택/지속턴/소모 이벤트가 클라이언트 간 불일치 위험.
2. **상태 저장 구조**
   - 기존 세이브 포맷이 단순 수치 중심일 경우 신규 상태 직렬화 충돌 가능.
3. **UI 표시**
   - 상태 아이콘/툴팁/중첩 수/남은 턴 시각화 부재 시 체감 품질 급락.
4. **전투 로그**
   - 다중 effects 처리 시 로그 순서 혼란 가능(적용/저항/해제 이벤트 구분 필요).
5. **밸런스**
   - 취약·관통·표식 조합의 폭딜 메타 편향 위험.
6. **기존 세이브/방 데이터 호환**
   - 구버전 룸/플레이어 데이터 로드 시 기본값 마이그레이션 필수.

## 8) 안전한 PR 분리안

- **PR1: 상태효과 엔진 추가**
  - 상태 정의, 적용/해제 파이프라인, 로그 이벤트 스키마
- **PR2: 검사/마법사 리워크**
  - 신규 상태 3~5개 활용, 초기 밸런스 값 적용
- **PR3: 도적/탱커 리워크**
  - 출혈/보호막 축 완성
- **PR4: 광전사/성직자/궁수/주술사 리워크**
  - 남은 역할군 전면 반영
- **PR5: 밸런스 조정**
  - 데이터 기반 수치 튜닝
- **PR6: UI 태그/설명 개선**
  - 상태 아이콘, 툴팁, 전투 로그 가독성 개선
